## بحرانی (CRITICAL)

### ۱. کتابخانه Chart.js موجود نیست - نمودارها کار نمی‌کنن
- **فایل:** `src/dashboard.js` خطوط ۱۶۴، ۱۸۷، ۲۰۳
- **توضیح:** `Chart` به صورت global استفاده شده ولی هیچجا import یا load نشده. نمودارهای داشبرد (دایره‌ای وضعیت مشتریان، دایره‌ای فروش، میله‌ای محصولات) همه خطا میدن.
- **راه‌حل:** اضافه کردن `<script src="https://cdn.jsdelivr.net/npm/chart.js">` به `index.html` یا نصب `chart.js` در package.json

### ۲. کتابخانه XLSX/SheetJS موجود نیست - ایمپورت/اکسپورت کار نمی‌کنه
- **فایل:** `src/import-export.js` خطوط ۸۴، ۹۲-۹۴، ۱۵۵، ۳۱۱
- **توضیح:** `XLSX` به صورت global استفاده شده ولی load نشده. تمام عملیات ایمپورت و اکسپورت اکسل خطا میدن.
- **راه‌حل:** اضافه کردن `<script src="https://cdn.jsdelivr.net/npm/xlsx/dist/xlsx.full.min.js">` به `index.html`

### ۳. کتابخانه jalaliDatepicker JS آن load نشده
- **فایل:** `src/main.js` خط ۲۰۷
- **توضیح:** فقط CSS کتابخانه link شده ولی JS آن load نشده. `jalaliDatepicker.startWatch()` خطا میده و کل `init()` crash می‌کنه.
- **راه‌حل:** اضافه کردن `<script src="https://unpkg.com/@majidh1/jalalidatepicker/dist/jalalidatepicker.min.js"></script>` به `index.html`

### ۴. تابع `showToast` در handler های inline HTML قابل دسترسی نیست
- **فایل:** `index.html` خط ۹۰
- **توضیح:** پروفایل dropdown از `onclick="showToast('پروفایل - به زودی')"` استفاده می‌کنه ولی `showToast` در scope ماژول ES هست نه global. خطای `ReferenceError` میده.
- **راه‌حل:** تغییر به `onclick="window.appShowToast('پروفایل - به زودی')"` و export کردن از main.js

## بالا (HIGH)

### ۵. `togglePermCheckbox` بدون پیشوند `window.` در HTML تولید شده
- **فایل:** `src/auth.js` خط ۱۸۳
- **توضیح:** چک‌باکس‌های دسترسی از `onchange="togglePermCheckbox(this)"` استفاده می‌کنن ولی این تابع در scope ماژول هست. خطای `ReferenceError` میده.
- **راه‌حل:** تغییر به `onchange="window.appTogglePermCheckbox(this)"`

### ۶. بازخورد بصری چک‌باکس‌های دسترسی کار نمی‌کنه
- **فایل:** `src/auth.js` خطوط ۱۸۳، ۲۳۱-۲۳۳
- **توضیح:** تابع `togglePermCheckbox` خالیه. وقتی کاربر چک‌باکس رو تغییر میده، رنگ پس‌زمینه تغییر نمی‌کنه.

### ۷. آسیب‌پذیری XSS در جدول مشتریان
- **فایل:** `src/customers.js` خطوط ۵۰-۱۱۶
- **توضیح:** فیلدهای `name`، `platformId`، `phone`، `notes`، `advisor` بدون `escapeHtml()` در innerHTML قرار می‌گیرن. اگه کاربری محتوای مخرب وارد کنه، اجرا میشه.
- **راه‌حل:** اضافه کردن `escapeHtml()` به تمام فیلدهای کاربر

### ۸. آسیب‌پذیری XSS در پنل جزئیات مشتری
- **فایل:** `src/customers.js` خطوط ۳۴۷-۴۶۳
- **توضیح:** فیلدهای `platformId`، `name`، `phone`، `notes` و داده‌های timeline بدون escape رندر میشن.

### ۹. آسیب‌پذیری XSS در جداول داشبرد
- **فایل:** `src/dashboard.js` خطوط ۱۱۵-۱۳۶
- **توضیح:** نام مشتریان و platformId در لیست‌های عقب افتاده/نزدیک بدون escape رندر میشن.

### ۱۰. آسیب‌پذیری XSS در جدول فروش‌ها
- **فایل:** `src/sales.js` خطوط ۱۰۰-۱۳۴، ۱۶۲-۱۹۶
- **توضیح:** `customerName`، `productName`، `customerPhone` بدون escape در innerHTML قرار می‌گیرن.

### ۱۱. آسیب‌پذیری XSS در جدول پیگیری‌ها
- **فایل:** `src/followups.js` خط ۵۰
- **توضیح:** `f.notes` بدون escape رندر میشه.

### ۱۲. عملیات async دیتابیس بدون `await` در تبدیل LD↔CS (خطر از دست رفتن داده)
- **فایل:** `src/customers.js` خطوط ۲۷۴-۲۷۶
- **توضیح:** سه عملیات async (`saveCustomerToDB`، `updateFollowupsCustomerId`، `saveSetting`) بدون `await` اجرا میشن. اگه خطا بدن، داده‌ها از همگام خارج میشن.

### ۱۳. عملیات async بدون `await` در setNextFollowup / clearNextFollowup
- **فایل:** `src/customers.js` خطوط ۴۸۰، ۴۹۲
- **توضیح:** `saveCustomerToDB()` بدون `await` صدا زده میشه. خطاهای دیتابیس سکوت میشن.

### ۱۴. عملیات async بدون `await` در توابع ایمپورت
- **فایل:** `src/import-export.js` خطوط ۲۶۰، ۴۲۴
- **توضیح:** در `doImport` و `doSalesImport`، `saveCustomerToDB` در حلقه forEach بدون await اجرا میشه.

### ۱۵. رمز عبور پیش‌فرض admin hardcoded شده
- **فایل:** `src/auth.js` خط ۴۶
- **توضیح:** رمز `admin123` hardcoded شده و هیچ مکانیزم تغییر اجباری رمز وجود نداره.

### ۱۶. هش رمز عبور بدون salt (SHA-256)
- **فایل:** `src/auth.js` خطوط ۸-۱۴
- **توضیح:** رمزها با SHA-256 ساده بدون salt هش میشن. در صورت نشت دیتابیس، در برابر rainbow table آسیب‌پذیره.
- **راه‌حل:** استفاده از bcrypt یا PBKDF2

### ۱۷. session در localStorage بدون زمان انقضا
- **فایل:** `src/utils.js` خطوط ۱۵۰-۱۶۶
- **توضیح:** session بدون تاریخ انقضا ذخیره میشه. تا زمان logout نامحدود باقی می‌مونه.

## متوسط (MEDIUM)

### ۱۸. بررسی دسترسی برای دکمه‌های عملیات وجود نداره
- **فایل:** `index.html`، `customers.js`، `followups.js`
- **توضیح:** دکمه‌های افزودن، حذف، ایمپورت، اکسپورت همیشه نمایش داده میشن حتی اگه کاربر دسترسی نداشته باشه:
  - `customers_add` → دکمه "+ مشتری جدید"
  - `customers_delete` → دکمه حذف در جدول
  - `customers_import` → دکمه ایمپورت
  - `customers_export` → دکمه‌های خروجی
  - `followups_add` → دکمه "+ پیگیری جدید"
  - `followups_delete` → دکمه حذف پیگیری
  - `followups_export` → دکمه‌های خروجی
  - `sales_import` → دکمه ایمپورت فروش
  - `sales_export` → دکمه‌های خروجی فروش

### ۱۹. تابع jalaliAddDays سال کبیسه رو در نظر نمی‌گیره
- **فایل:** `src/utils.js` خطوط ۹۰-۹۷
- **توضیح:** ماه اسفند همیشه ۲۹ روز فرض شده ولی در سال کبیسه ۳۰ روزه.

### ۲۰. حذف مشتری قبل از تأیید دیتابیس از حافظه حذف میشه
- **فایل:** `src/customers.js` خطوط ۳۱۴-۳۲۱
- **توضیح:** مشتری از حافظه حذف میشه ولی `deleteCustomerToDB` بدون await اجرا میشه. اگه دیتابیس خطا بده، مشتری در حافظه نیست ولی در دیتابیس هست.

### ۲۱. شناسایی پیگیری‌ها بر اساس index آرایه شکننده‌ست
- **فایل:** `src/followups.js` خطوط ۳۹، ۷۷، ۱۰۸
- **توضیح:** پیگیری‌ها بدون primary key پایدار شناسایی میشن. اگه آرایه تغییر کنه، index نادرست میشه.

### ۲۲. حذف پیگیری تمام پیگیری‌های مشتری رو حذف و دوباره insert می‌کنه
- **فایل:** `src/data.js` خطوط ۱۴۳-۱۶۰
- **توضیح:** برای حذف یک پیگیری، تمام پیگیری‌های مشتری حذف و بقیه دوباره insert میشن. برای مشتریان با پیگیری زیاد کند و ناکارآمد.

### ۲۳. `sortSales` فیلترها و جستجوی `renderSales` رو رعایت نمی‌کنه
- **فایل:** `src/sales.js` خطوط ۱۴۳-۱۹۷
- **توضیح:** بعد از مرتب‌سازی، فروش‌هایی که قبلاً فیلتر شده بودن ممکنه دوباره نمایش داده بشن.

### ۲۴. badge شناسه پیگیری همیشه استایل LD نشون میده
- **فایل:** `src/followups.js` خط ۴۴
- **توضیح:** تمام badge‌های شناسه مشتری در جدول پیگیری‌ها کلاس `id-ld` دارن حتی اگه مشتری CS باشه.

### ۲۵. فیلد نام مشتری اعتبارسنجی نداره
- **فایل:** `src/customers.js` خط ۲۳۲
- **توضیح:** نام اختیاریه ولی اعتبارسنجی طول یا کاراکتر خاص نداره.

### ۲۶. `saveProductField` بدون `await` اجرا میشه
- **فایل:** `src/customers.js` خطوط ۵۹۱، ۵۹۴-۵۹۶
- **توضیح:** تغییر قیمت/بیعانه محصولات `saveCustomerToDB` رو بدون await صدا میزنه.

### ۲۷. `saveCustomerToDB` در افزودن/حذف محصولات بدون `await`
- **فایل:** `src/customers.js` خط ۵۷۰
- **توضیح:** تابع `setProducts` که `saveCustomerToDB` رو صدا میزنه async نیست.

### ۲۸. فیلتر تاریخ داشبرد فروش‌های بدون تاریخ تسویه رو حذف می‌کنه
- **فایل:** `src/dashboard.js` خط ۸۳
- **توضیح:** وقتی فیلتر تاریخ فعاله، فروش‌هایی که `settlementDate` ندارن حذف میشن.

### ۲۹. `confirm()` مرورگر برای حذف کاربر به جای مودال سفارشی
- **فایل:** `src/auth.js` خط ۱۵۸
- **توضیح:** حذف کاربر از `confirm()` مرورگر استفاده می‌کنه ولی حذف مشتری/پیگیری از مودال سفارشی. عدم سازگاری UX.

### ۳۰. مودال لاگین ممکنه محتوای اصلی رو نشون بده قبل از بررسی session
- **فایل:** `index.html` خط ۱۷، `src/main.js` خطوط ۱۷۹-۱۸۶
- **توضیح:** overlay لاگین hidden پیش‌فرضه ولی `checkSession` بعد از `loadData` اجرا میشه. در زمان لود، محتوای اصلی ممکنه دیده بشه.

### ۳۱. فیلد رمز عبور بعد از خطا لاگین پاک نمیشه
- **فایل:** `src/auth.js` خطوط ۸۰-۸۴
- **توضیح:** بعد از خطای لاگین، رمز در فیلد باقی می‌مونه.

## پایین (LOW)

### ۳۲. کد مرده: متغیرهای استفاده نشده در main.js
- **فایل:** `src/main.js` خطوط ۱۶۵-۱۶۶
- **توضیح:** `origDoImport` و `origDoSalesImport` تعریف شدن ولی هیچجا استفاده نشدن.

### ۳۳. تابع `toJalali` تکراری
- **فایل:** `src/utils.js` خط ۴۳ و `src/customers.js` خط ۵۱۹
- **توضیح:** تابع تبدیل میلادی به شمسی در دو فایل پیاده‌سازی شده.

### ۳۴. تابع `escapeHtml` گیومه تکی رو escape نمی‌کنه
- **فایل:** `src/utils.js` خطوط ۲۷-۳۰
- **توضیح:** فقط `&`، `<`، `>`, `"` رو escape می‌کنه نه `'` رو.

### ۳۵. `formatNumber` برای مقدار `""` رشته خالی برمی‌گردونه
- **فایل:** `src/utils.js` خطوط ۱۱-۱۶
- **توضیح:** edge case ولی ممکنه caller ها رو گیج کنه.

### ۳۶. Toast ها روی هم اعمال نمیشن - پیام‌های سریع جایگزین میشن
- **فایل:** `src/utils.js` خطوط ۳۲-۳۷
- **توضیح:** فقط یک toast element وجود داره. اگه سریع پیام ارسال بشه، پیام قبلی از بین میره.

### ۳۷. گرید نمودارهای داشبرد در صفحه‌های کوچک ریسپانسیو نیست

### ۳۸. دکمه "+ مشتری جدید" بدون بررسی دسترسی `customers_add`
- **فایل:** `index.html` خط ۲۱۵
- **توضیح:** دکمه همیشه نمایش داده میشه.

### ۳۹. موقعیت dropdown پروفایل در RTL

### ۴۰. عدم سازگاری await در init()

### ۴۱. مرتب‌سازی مشتریان آرایه رو in-place تغییر میده

### ۴۲. نشانگر loading در زمان لود اولیه وجود نداره

## اولویت‌بندی پیشنهادی برای رفع

### فوری (قبل از پروداکشن)
1. اضافه کردن کتابخانه‌های Chart.js، XLSX، jalaliDatepicker به `index.html`
2. رفع خطای `showToast` در HTML
3. رفع خطای `togglePermCheckbox` در auth.js
4. اضافه کردن `escapeHtml` به تمام فیلدهای کاربر در innerHTML

### نزدیک (هفته آینده)
5. اضافه کردن await به تمام عملیات async دیتابیس
6. اضافه کردن بررسی دسترسی به دکمه‌های عملیات
7. بهبود هش رمز عبور (bcrypt)
8. اضافه کردن زمان انقضا به session

### بعدی (ماه آینده)
9. اضافه کردن primary key به پیگیری‌ها
10. بهینه‌سازی حذف/ویرایش پیگیری‌ها
11. اضافه کردن loading indicator
12. رفع باگ‌های کوچک
