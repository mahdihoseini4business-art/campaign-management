## بحرانی (CRITICAL)

### C1 — خطای Runtime: import نشده `updateFollowupInDB`
- **فایل:** `src/followups.js` خط ۱۲۳
- **توضیح:** تابع `updateFollowupInDB` صدا زده میشه ولی import نشده. باعث `ReferenceError` موقع ویرایش پیگیری میشه.

### C2 — خطای Runtime: import نشده `escapeHtml` در فروش‌ها
- **فایل:** `src/sales.js` خط ۸۹
- **توضیح:** `escapeHtml` در `renderSalesRows()` استفاده شده ولی import نشده. باعث `ReferenceError` موقع رندر فروش‌ها میشه.

### C3 — لایه دیتابیس تمام خطاهای DB رو سکوت می‌کنه
- **فایل:** `src/data.js` خطوط ۸۳، ۹۵، ۱۱۳، ۱۲۷، ۱۳۶، ۱۴۶، ۱۵۵
- **توضیح:** تمام توابع ذخیره‌سازی دیتابیس خطاها رو فقط `console.error` می‌کنن و throw نمی‌کنن. اگه ذخیره ناموفق باشه، UI "ذخیره شد" نشون میده ولی داده ذخیره نشده.

## بالا (HIGH)

### H1 — بعد از ایمپورت، لیست مشتریان و فروش‌ها رفرش نمیشه
- **فایل:** `src/import-export.js` خطوط ۲۶۴-۲۶۵ و ۴۲۹-۴۳۳
- **توضیح:** بعد از `doImport()` و `doSalesImport()` تابع render صدا زده نمیشه. کاربر باید رفرش کنه.

### H2 — `addUser` شکست ذخیره رو نادیده می‌گیره
- **فایل:** `src/auth.js` خطوط ۱۵۵-۱۶۳
- **توضیح:** `saveUser()` در صورت خطا false برمی‌گردونه ولی `addUser()` بررسی نمیکنه. همیشه toast موفقیت نشون میده.

### H3 — `sortState` بین مرتب‌سازی مشتریان و پیگیری‌ها مشترکه
- **فایل:** `src/main.js` خط ۳۵
- **توضیح:** یک آبجکت `sortState` مشترک بین `sortCustomers` و `sortFollowups` استفاده میشه. مرتب‌سازی پیگیری‌ها state مشتریان رو تغییر میده.

### H4 — مشتری جدید فیلد `nextFollowupDate` نداره
- **فایل:** `src/customers.js` خط ۲۵۵
- **توضیح:** موقع ایجاد مشتری جدید، فیلد `nextFollowupDate` به آبجکت اضافه نشده.

### H5 — بررسی سمت سرور برای session وجود نداره
- **فایل:** `src/auth.js` خطوط ۱۱۰-۱۱۸
- **توضیح:** `checkSession()` فقط از localStorage میخونه و با سرور چک نمیکنه. اگه ادمین دسترسی کاربری رو حذف کنه، تا logout تاثیر نداره.

### H6 — حذف مشتری با شکست سکوت دیتابیس از همگام خارج میشه
- **فایل:** `src/data.js` خطوط ۹۰-۹۶
- **توضیح:** `deleteCustomerFromDB` اگه خطا بده، مشتری از حافظه حذف شده ولی در دیتابیس باقی میمونه.

### H7 — تبدیل LD→CS ممکنه داده از دست بده
- **فایل:** `src/customers.js` خطوط ۲۶۹-۲۸۰
- **توضیح:** موقع تبدیل، سه عملیات async بدون مدیریت خطا اجرا میشن. اگه هر کدوم خطا بده، داده‌ها از همگام خارج میشن.

## متوسط (MEDIUM)

### M1 — نمک رمز عبور قابل پیش‌بینیه
- **فایل:** `src/auth.js` خط ۱۲
- **توضیح:** نمک PBKDF2 بر اساس نام کاربری ساخته میشه. نمک باید تصادفی و منحصربفرد باشه.

### M2 — رمز عبور پیش‌فرض admin hardcoded شده
- **فایل:** `src/auth.js` خط ۵۸
- **توضیح:** رمز `admin123` در کد وجود داره. مکانیزم اجبار تغییر رمز وجود نداره.

### M3 — محدودیت تعداد تلاش‌های لاگین وجود نداره
- **فایل:** `src/auth.js` خطوط ۷۳-۱۰۳
- **توضیح:** هیچ محدودیتی برای تعداد تلاش‌های لاگین وجود نداره. امکان brute force وجود داره.

### M4 — XSS از طریق نام کاربری در onclick
- **فایل:** `src/auth.js` خط ۲۱۴
- **توضیح:** نام کاربری بدون escape در attribute HTML قرار میگیره. اگه نام کاربری شامل `'` باشه، XSS ممکنه.

### M5 — بررسی دسترسی فقط سمت کلاینته
- **فایل:** `index.html` خطوط ۲۱۵-۲۱۸
- **توضیح:** بررسی دسترسی دکمه‌ها فقط در JavaScript انجام میشه. کاربر میتونه از console مرورگر مستقیماً توابع رو صدا بزنه.

### M6 — `init` بدون مدیریت خطای سطح بالاست
- **فایل:** `src/main.js` خط ۲۱۲
- **توضیح:** اگه `seedAdmin` یا `loadData` خطا بدن، برنامه سکوت میشه و صفحه خالی میمونه.

### M7 — ذخیره ترتیبی در ایمپورت (کندی)
- **فایل:** `src/import-export.js` خطوط ۲۵۹-۲۶۲
- **توضیح:** هر مشتری جداگانه ذخیره میشه. برای حجم زیاد بسیار کنده. باید batch upsert استفاده بشه.

### M8 — ایمپورت فروش بیش از حد مشتری ذخیره می‌کنه
- **فایل:** `src/import-export.js` خط ۴۲۳
- **توضیح:** تمام مشتریانی که محصول دارن ذخیره میشن، نه فقط مشتریان تغییر یافته.

### M9 — ویرایش پیگیری بر اساس index آرایه شکننده‌ست
- **فایل:** `src/followups.js` خطوط ۳۹، ۵۳-۵۴
- **توضیح:** `realIndex` از `indexOf` محاسبه میشه. اگه آرایه تغییر کنه، index نادرست میشه.

### M10 — مودال حذف مشترک onclick handler داره
- **فایل:** `src/customers.js` خط ۳۱۸، `src/followups.js` خط ۱۴۵
- **توضیح:** هر دو تابع `onclick` روی `#deleteConfirmBtn` تنظیم میکنن. ممکنه overwrite بشه.

### M11 — قیمت محصولات به صورت string ذخیره میشه
- **فایل:** `src/customers.js` خطوط ۵۸۴-۶۲۸
- **توضیح:** قیمت‌ها string ذخیره میشن ولی در جاهای مختلف با `parseFloat` تبدیل میشن. این عدم سازگاری مشکل‌ساز میشه.

### M12 — `user-scalable=no` مانع زوم موبایل میشه
- **فایل:** `index.html` خط ۵
- **توضیح:** جلوی زوم کاربران موبایل رو می‌گیره. نقض WCAG 2.1.

### M13 — ایمپورت فروش چک duplicate بیش از حد سخت‌گیرانه‌ست
- **فایل:** `src/import-export.js` خطوط ۴۱۳-۴۱۶
- **توضیح:** اگه مشتری دوبار محصول رو با قیمت و وضعیت یکسان بخره، بار دوم رد میشه.

### L11 — `autofocus` در فرم لاگین کار نمی‌کنه

### L13 — تبدیل digit جهانی ممکنه با ویجت‌ها تداخل کنه

### L14 — `noSetCount` در داشبرد فیلتر تاریخ رو رعایت نمی‌کنه
